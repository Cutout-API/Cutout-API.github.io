The Design of Cutout

There are three main components to the Cutout API: SVG Parsing, Mesh Creation, and Mesh Rendering. Each of these components resides in its own library. The Cutout API allows developers to input an SVG image to the mesh creation library where it will be parsed and converted into a 2D mesh. This mesh data can then be passed to the rendering library to properly display it in a 3D scene.

The Parsing Process

The Parsing process of the Cutout API involves parsing SVG and RAW files for use in the Mesh Creation process of the API. A developer would input an SVG or RAW file that includes the 2D vector asset that the developer wants to render on screen. The Parser process of the API then outputs optimized polygon path data that the Mesh Creation process can use to generate its meshes.

The Mesh Creation Process

The Mesh Creation process of the Cutout API generates an optimal 2D triangular mesh that accurately represents the input polygon path data for use by the Renderer process. Using the polygon path data retrieved from file, the Mesh Creation is capable of things like determining the correct number of points that should be rendered for a simple curve.

The Rendering Process

The Rendering portion of the API makes function calls to next level graphics APIs in order to render the meshes generated by Mesh Creator. This pipeline can also include runtime optimizations. For example, the rendering pipeline can determine the level of detail to render depending on how far away an object is on the screen.
