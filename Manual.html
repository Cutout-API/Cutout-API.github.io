The User Manual

This is a guide that explains how the API can be used by developers once the API is released. This document will be changing in the future as we prepare to release the Cutout API. At the moment we assume the use of Visual Studios, but the steps can be generalized to most other IDE's.

Currently this manual is just meant as a conceptualization of how things will work for interested parties. As we get closer to release we will update this to include detailed guides.

How To Use The API

Their will be 3 primary ways of using the API:

  • Statically linking to the entire Cutout Library
  • As a standalone entity (basically a dll)
  • Linking to individual libraries within Cutout

How To Statically Link To Cutout

Linking to the Cutout library is fairly straight foward. The first step is to download the code available here.

Next you will want to add the folder containing Cutout to your VS solution as a seperate project. This will hopefully automatically set up build information. Otherwise you will need to set up the Cutout projects to build as static libraries. Once Cutout is building and outputing the library files you need to tell your other projects to use the Cutout library file during linking. This can be done through the settings menu of your project.

Now to use Cutout you just need to include the headers containing the functionality you wish to use and implement your code using the Cutout API.

Documentation of all the classes and functions that are available through the Cutout API can be found here

How To Dynamically Link To Cutout

We will not be providing DLL's for Cutout at release. What we plan on providing is code that you can easily compile to form your own DLL that you can then use in your projects.

The first step is to download the code available here.

Once downloaded the code comes with a VS solution that you can open up. Once open if you click build VS will generate the dynamic libraries for you.

These DLL's will work like any other. You can attach to them at link or runtime and access the libraries using standard methods for dynamic libraries.

In the future we will also provide functionality for shared libraries in linux.

How To Link To Individual Libraries Of Cutout

The Cutout API actually contains 3 seperate libraries as can be seen here.

Depending on how you wish to use the library you might want to just use one or two of the modules and not the others. Like lets say you want to use the Mesh Generation portion and create your own parser and renderer. We designed Cutout to allow for this type of plug and play scenerio.

To link to an individual modules of Cutout you just need its code and then integrate it into your project using one of the other two methods. It'll work the same and only have the features on the module's you chose to integrate with.
